# ScienceAgentBench: Toward Rigorous Assessment of Language Agents for Data-Driven Scientific Discovery

- **저자**: Ziru Chen, Shijie Chen, Yuting Ning, Qianheng Zhang, Boshi Wang, Botao Yu, Yifei Li, Zeyi Liao, Chen Wei, Zitong Lu, Vishal Dey, Mingyi Xue, Frazier N. Baker, Benjamin Burns, Daniel Adu-Ampratwum, Xuhui Huang, Xia Ning, Song Gao, Yu Su, Huan Sun
- **소속**: Ohio State University, UW-Madison
- **발표**: ICLR 2025
- **DOI**: [10.48550/arXiv.2410.05080](https://doi.org/10.48550/arXiv.2410.05080)

---

## Essence (본질)

LLM 기반 언어 에이전트가 데이터 기반 과학적 발견(data-driven scientific discovery)을 실제로 자동화할 수 있는지를 엄밀하게 평가하기 위한 벤치마크 **ScienceAgentBench**를 제안한다. 4개 분야(생물정보학, 전산화학, 지리정보과학, 심리학/인지신경과학)의 44편 피어리뷰 논문에서 추출한 102개 과제를 포함하며, 각 과제의 목표 출력을 독립 실행 가능한 Python 프로그램으로 통일하여 체계적으로 평가한다. 최고 성능 에이전트(Claude-3.5-Sonnet + self-debug)도 32.4%의 과제만 독립적으로 해결할 수 있어, 현재 LLM 에이전트의 과학 자동화 능력에 대한 과장된 주장에 경종을 울린다.

## Motivation (동기)

최근 LLM 기반 에이전트가 과학 연구의 전 과정(아이디어 생성 -> 실험 -> 논문 작성)을 자동화할 수 있다는 대담한 주장들이 등장하였으나, 개별 과제 수준에서의 엄밀한 평가가 부족했다. 기존 벤치마크(SWE-Bench, MLAgentBench 등)는 소프트웨어 엔지니어링 위주이거나, 실제 과학 연구의 다양성과 복잡성을 반영하지 못하며, 데이터 오염(data contamination)과 에이전트의 편법(shortcut) 문제에 대한 대응이 미흡했다. 과학 연구 자동화를 주장하기 전에, 워크플로우의 개별 단계에서 에이전트의 실질적 능력을 먼저 검증해야 한다는 문제의식에서 출발한다.

## Achievement (성과)

1. **102개 고품질 과학 과제 벤치마크 구축**: 44편 피어리뷰 논문에서 추출하고 9명의 분야 전문가(SME)가 검증한 과제를 제공. 데이터 처리, 모델 개발, 데이터 분석, 시각화 등 다양한 하위 과제를 포괄.
2. **체계적 평가 프레임워크**: Valid Execution Rate(VER), Success Rate(SR), CodeBERTScore(CBS), Cost의 4가지 메트릭과 5단계 루브릭 기반 인간 평가를 결합.
3. **데이터 오염 및 편법 방지 전략**: 테스트셋에서 데이터 포인트를 무작위 제거하고, 특성 변환을 적용하여 기존 학습 데이터 유출과 에이전트 편법을 효과적으로 차단.
4. **포괄적 실험 결과**: 5개 오픈/프로프라이어터리 LLM x 3개 프레임워크(Direct Prompting, OpenHands CodeAct, Self-Debug) 조합을 평가. 최고 성능은 Claude-3.5-Sonnet + Self-Debug로 SR 32.4%(지식 제공 시 34.3%). OpenAI o1-preview는 42.2%까지 도달하나 비용이 10배 이상.

## How (방법론)

- **과제 구성**: 각 과제는 (a) 과제 지시문, (b) 데이터셋 정보, (c) 전문가 제공 지식(선택), (d) 주석 프로그램의 4개 요소로 구성. 목표 출력은 항상 자체 실행 가능한 Python 프로그램.
- **평가 파이프라인**: conda 환경을 자동 구성하여 생성된 프로그램을 실행. pipreqs로 패키지 분석 후 pip-tools로 환경 설정. 4가지 자동 메트릭 + GPT-4o 기반 시각화 판정 + 루브릭 기반 인간 평가.
- **3개 에이전트 프레임워크 비교**:
  - **Direct Prompting**: LLM에 직접 과제를 제시하여 프로그램 생성
  - **OpenHands CodeAct**: Python 인터프리터, bash, 웹 브라우저 등 다양한 도구를 갖춘 범용 에이전트
  - **Self-Debug**: 생성된 프로그램을 실행하고 오류 피드백을 반영하여 반복 개선
- **안정적 평가**: 각 과제를 3회 독립 실행하여 SR > VER > CBS > Cost 순으로 최선의 결과를 선택.

## Originality (독창성)

- **과학 연구 워크플로우의 개별 단계를 분리하여 평가**한다는 접근이 독창적. End-to-end 자동화를 주장하기 전에 구성 요소별 능력을 엄밀히 검증해야 한다는 방법론적 주장이 설득력 있음.
- 데이터 오염 방지를 위한 **테스트셋 수정 전략**(무작위 제거 + 특성 변환)은 기존 벤치마크에서 간과되던 문제를 실질적으로 해결.
- **이질적 과학 데이터**(세포 이미지, 분자 구조, 지리 지도, EEG 시계열)를 포괄하는 다학제적 벤치마크로, 기존 ML/코드 벤치마크와 차별화.
- Self-Debug가 OpenHands CodeAct보다 더 높은 SR을 17배 낮은 비용으로 달성한다는 발견은 에이전트 설계에 대한 중요한 시사점을 제공.

## Limitation & Further Study (한계 및 향후 연구)

1. **과제 규모**: 102개 과제는 4개 분야를 다루기에는 제한적이며, 물리학, 재료과학, 지구과학 등 추가 분야로의 확장이 필요.
2. **평가 메트릭의 한계**: 루브릭 기반 평가가 gold program 기준으로 작성되어, 동등하게 유효하지만 다른 접근법(예: 랜덤 포레스트 vs 신경망)을 과소평가할 수 있음.
3. **정적 벤치마크**: 과학 연구는 동적이고 반복적인 과정이나, 벤치마크는 일회성 코드 생성만 평가. 가설 수립, 실험 설계, 결과 해석 등의 상위 인지 과정은 포함하지 않음.
4. **LLM 발전 속도**: 벤치마크 구축 이후 등장하는 새로운 모델(GPT-4o 이후 세대)이 빠르게 성능을 향상시킬 수 있어, 지속적 업데이트가 필요.
5. **전문가 지식 활용의 비효율성**: 전문가 제공 지식이 SR을 소폭 향상시키지만 VER은 오히려 감소하는 현상에 대한 심층 분석과 해결책이 필요.
6. **비용-성능 트레이드오프**: o1-preview가 높은 SR을 달성하나 비용이 10배 이상 증가하는 문제에 대한 효율적 추론 전략 연구가 필요.

## Evaluation (총평)

ScienceAgentBench는 LLM 에이전트의 과학 연구 자동화 능력을 현실적으로 평가하기 위한 **시의적절하고 엄밀한 벤치마크**이다. 과학 연구의 개별 단계별 능력을 분리하여 측정한다는 접근법은 방법론적으로 건전하며, 데이터 오염 방지와 편법 차단 전략은 벤치마크의 신뢰성을 높인다. 최고 성능 에이전트도 전체 과제의 1/3 수준만 해결할 수 있다는 결과는, 현재의 AI 과학 자동화 주장에 대한 건설적인 반론을 제공한다. 다만, 과학 연구의 본질인 가설 생성과 창의적 사고는 평가 범위 밖이며, 벤치마크의 지속적 확장과 업데이트가 과제로 남는다. ICLR 2025에 채택된 것이 당연한 수준의 완성도를 갖춘 연구이다.

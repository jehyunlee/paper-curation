Scaling Reproducibility: An AI-Assisted Workflow for Large-Scale Reanalysis

저자: Yiqing Xu, Leo Yang Yang | 날짜: 2026-02-17 | arXiv: 2602.16733 | 분야: econ.EM

Figure 1

Essence

실증 사회과학 연구의 대규모 재현성 검증을 자동화하는 에이전트 AI 워크플로우를 개발하고 평가한다. 핵심 설계 원칙은 "과학적 추론과 계산 실행의 분리"로, 인간 연구자가 진단 템플릿(estimand, estimator, robustness check)을 사전 설계하면, AI 워크플로우가 복제 자료의 획득, 환경 재구성, 코드 실행, 분석 데이터셋 추출, 표준화 진단 보고서 생성을 자동으로 수행한다. 92편의 도구변수(IV) 연구(215개 사양)에 대해 전체 87%, 접근 가능한 자료 조건하 100%의 end-to-end 재현 성공률을 달성했다.

Motivation

  • 알려진 사실: 재현성은 연구 신뢰도의 근간이며, 주요 저널들이 데이터/코드 공개를 요구하고 있음. 그러나 복제 패키지의 소프트웨어 환경, 디렉토리 구조, 문서화 품질이 크게 달라 대규모 재현은 여전히 비용이 높음
  • 격차(Gap): 저자들의 선행 연구(Lal et al. 2024; Chiu et al. 2023 등)에서 수십 편의 논문을 수동으로 재현하는 데 각각 3~4년이 소요됨. 이 시간의 대부분이 방법론이 아닌 코드 해독, 환경 재구성, 이질적 자료 조화에 소비
  • 접근법: Claude Code Skills 기반 3계층 아키텍처(LLM 오케스트레이터 / 스킬 기술+지식 베이스 / 결정론적 에이전트 코드)로, 적응적 조정과 결정론적 계산을 분리. 반복적 실패 패턴을 버전 관리된 해결 규칙으로 축적

Achievement

  1. 87% end-to-end 재현 성공률: 92편 IV 연구 중 80편 자율 성공 (실패 12편은 자료 미공개). 접근 가능 자료 조건에서 논문 수준/사양 수준 모두 100% 성공
  2. 215개 사양에 대한 완전 재현: 논문당 최대 3개 2SLS 사양을 자동으로 식별, 실행, 진단. 기존 Lal et al.(2024)의 70개 사양에서 215개로 3배 확장
  3. 7단계 모듈 파이프라인: Profiler(메타데이터/사양 추출) -> Librarian(자료 다운로드) -> Janitor(코드 정리) -> Runner(실행/데이터 추출) -> Skeptic(진단 분석) -> Journalist(보고서 생성)의 체계적 구조
  4. 적응적 지식 축적: 64개 해결 패턴이 10개 실패 클래스(경로/환경, 소프트웨어/언어, 데이터 포맷, 변수명 불일치, Stata 구문 변형 등)에 걸쳐 축적. 초기 63%에서 92.5%로 성공률 향상
  5. 실증적 재발견: 자동 파이프라인으로 재현한 215개 사양에서도 Lal et al.(2024)의 핵심 발견(관측 연구에서 2SLS/OLS 불일치와 1단계 강도의 음의 상관)이 재확인됨

How

  • 3계층 아키텍처: Layer 1(Claude LLM 오케스트레이터: 태스크 라우팅, 오류 해석), Layer 2(SKILL.md: 에이전트별 인터페이스 계약 + 축적된 실패 패턴 지식), Layer 3(R/Stata/Python 결정론적 코드: 모든 수치 연산 수행)
  • 재현 기준: 조화된(harmonized) 파이프라인에서 생성한 2SLS 점추정치가 원저자 파이프라인의 추정치와 정확히 일치
  • 교차 언어 검증: Stata에서 실행한 결과를 R의 iv_robust()로 재추정하여 수치적 허용 오차 내 일치 확인
  • 진단 템플릿: ivDiag R 패키지 기반 -- 도구 강도(effective F), 견고한 추론(Anderson-Rubin, bootstrap, tF), 민감도 분석(jackknife), 2SLS-OLS 비교
  • 적용 데이터: APSR, AJPS, JOP 3개 저널의 2010-2025년 IV 연구 92편 (67편 기존 벤치마크 + 25편 신규)

Originality

"과학적 추론은 인간이, 계산 실행은 AI가"라는 분업 원칙을 대규모 재현성 검증에 체계적으로 적용한 최초의 연구이다. 특히 (1) 버전 관리된 적응적 학습 메커니즘(실패 패턴을 일반화 규칙으로 인코딩)으로 이질적 복제 패키지를 점진적으로 처리하면서도 수치적 결정성을 유지하는 설계가 독창적이며, (2) Stata/R/Python 다언어 코드베이스를 자동 파싱하고 교차 검증하는 실제 엔지니어링 성과가 인상적이다. 10개 클래스, 64개 해결 패턴의 상세한 실패 인벤토리 문서화는 후속 연구자에게 귀중한 자원이다.

Limitation & Further Study

저자들이 언급한 한계

  • 정치학 분야 IV 설계에 한정되며, 데이터/코드가 공개된 경우에만 작동
  • 새로운 통계 추정량이나 진단 도구를 제안하지 않음 -- 기존 템플릿의 실행 자동화에 집중
  • 불확실성 추정(bootstrap, jackknife)은 random seed 이식성 문제로 정확한 재현이 더 어려움
  • 신규 25편에 대해서는 출판 계수와 파이프라인 계수 간 1차 동등성이 독립 검증되지 않음

리뷰어 판단 아쉬운 점

  • 적응 과정의 인간 개입 범위가 불명확: "human-in-the-loop"로 실패 규칙을 승인한다고 하지만, 실제로 얼마나 많은 인간 시간이 투입되었는지 정량적으로 보고되지 않음. "자동"이라는 주장과 인간 검토 사이의 긴장이 존재
  • 일반화 가능성 미검증: 경제학의 DID, RCT, 구조 모형 등 다른 연구 설계로의 확장을 언급하지만 실제 실험은 미실시
  • 벤치마크에 대한 과적합 가능성: 64개 해결 규칙이 92편 코퍼스에 대해 반복적으로 개발되었으므로, 완전히 새로운 코퍼스에서의 성공률은 불확실
  • 비용 분석 부재: LLM API 호출 비용, 총 처리 시간 등 자동화의 경제적 이점에 대한 정량적 비교가 없음
  • AI의 방법론적 판단 위험: 사양 선택(어떤 ivreg2를 "primary"로 분류할지)에 LLM이 관여하는데, 이 판단의 정확도에 대한 체계적 평가가 부족

후속 연구

  • DID, TWFE, interaction model 등 다른 인과추론 설계로의 확장
  • 완전 자동 테이블/그림 재현 및 내적 일관성 검증
  • 학술지 투고/리뷰 과정 통합 가능성 탐색
  • 새로운 코퍼스(경제학, 심리학)에서의 일반화 성능 평가

Evaluation

Novelty: 4/5 Technical Soundness: 4/5 Significance: 5/5 Clarity: 5/5 Overall: 4/5

총평: 대규모 재현성 검증이라는 과학의 근본적 과제를 에이전트 AI 워크플로우로 체계적으로 해결한 매우 의미 있는 연구이다. "과학적 추론과 계산 실행의 분리"라는 설계 원칙이 명확하고, 92편/215개 사양에 대한 100% 조건부 성공률은 설득력 있는 실증이다. 특히 10개 클래스 64개 해결 패턴의 상세한 실패 인벤토리는 재현성 연구 분야에 대한 독립적인 기여이다. 다만 적응 과정의 인간 개입 정도와 새로운 코퍼스에서의 일반화 가능성이 더 명확해질 필요가 있다.

OpenFOAMGPT 2.0: end-to-end, trustworthy automation for computational fluid dynamics

Figure 1

Essence

자연어 질의로부터 전처리-시뮬레이션-후처리까지 완전 자동화된 end-to-end CFD 시뮬레이션을 수행하는 최초의 멀티 에이전트 프레임워크를 제안한다. 4개의 전문 에이전트(Pre-processing, Prompt Generation, OpenFOAMGPT, Post-processing)가 Claude-3.7-Sonnet을 기반으로 협력하며, Poiseuille 유동, 다공성 매질 유동, 공기역학 해석 등 다양한 사례에서 450회 이상의 시뮬레이션에서 100% 성공률과 재현성을 달성했다. "대화 기반 시뮬레이션(conversation-to-simulation)" 워크플로우를 통해 CFD의 진입 장벽을 크게 낮추면서도 수치적 엄밀성을 유지했다.

Motivation (동기)

  1. CFD의 높은 진입 장벽: 전산유체역학은 정밀한 수치 사양, 복잡한 형상 처리, 전문적인 물리 모델링, 특수한 결과 해석이 필요하여, 전문가가 아니면 접근이 어렵다.
  2. LLM-CFD 통합의 도전: LLM의 강력한 자연어 이해 능력에도 불구하고, OpenFOAM의 엄격한 구문 요구사항, 복잡한 파라미터 상호작용, 무결점(zero-tolerance) 시뮬레이션 환경에서의 신뢰성 확보가 핵심 과제였다.
  3. 기존 시스템의 부분적 커버리지: 선행 연구인 OpenFOAMGPT 1.0은 시뮬레이션 실행에 초점을 두었으나, 전처리와 후처리를 포함한 end-to-end 워크플로우는 구현하지 못했다.

Achievement (성과)

  • 100% 성공률 및 재현성: 5가지 사례 연구에서 총 455회 이상의 시뮬레이션을 수행하여, 단일/다중 케이스 파라미터 스터디 모두에서 100% 성공률과 완벽한 재현성을 달성했다.
  • 다양한 물리 현상 커버: 단상 Poiseuille 유동, 다상 Poiseuille 유동, 단상/다상 다공성 매질 유동, 오토바이 공기역학(난류) 등 광범위한 CFD 문제를 처리했다.
  • 자동 파라미터 스터디: 격자 독립성 연구, REV 크기 결정, 접촉각/유입 속도/점도비 변화 연구 등 복잡한 파라미터 스터디를 인간 개입 없이 자율적으로 수행했다.
  • 자동 해석적 검증: Post-processing Agent가 Poiseuille 유동의 해석해를 자발적으로 계산하여 수치해와 비교하는 등, 결과 검증까지 자동화했다.

How (방법론)

4개 전문 에이전트 구조

에이전트역할핵심 기능
**Pre-processing Agent**사용자 질의 분석단일/다중 케이스 판별, 메쉬 전략 선택 (blockMesh vs snappyHexMesh)
**Prompt Generation Agent**시뮬레이션별 프롬프트 생성파라미터 분해, Prompt Pool 관리, 케이스별 독립 프롬프트 생성
**OpenFOAMGPT**핵심 시뮬레이션 엔진설정 파일 생성 -> 자동 실행 -> 에러 기반 반복 수정 (self-correcting loop)
**Post-processing Agent**결과 시각화 및 분석Python 스크립트 자동 생성, Paraview VTK, 출판 품질 그래프

핵심 설계 결정

  • Temperature = 0: OpenFOAM 설정 파일의 엄격한 구문 요구사항에 대응하여, LLM의 "thinking" 기능을 비활성화하고 결정론적 출력을 보장
  • 케이스별 독립 프롬프트: 기존 파일을 수정하는 방식 대신, 각 시뮬레이션마다 새로운 완전한 프롬프트를 생성하여 점진적 수정의 오류 위험 제거
  • Docker 컨테이너: OpenFOAM v2406 Docker를 사용하여 호스트 시스템과 무관한 표준화된 실행 환경 제공
  • 자기 수정 루프: 시뮬레이션 실패 시 에러 로그를 LLM에 피드백하여 설정 파일을 반복적으로 개선

Originality (독창성)

  1. CFD 최초의 멀티 에이전트 end-to-end 프레임워크: 자연어 -> 전처리 -> 시뮬레이션 -> 후처리 -> 시각화/분석까지 전 과정을 자동화한 최초의 CFD 시스템이다.
  2. Zero-tolerance 환경에서의 100% 신뢰성: 과학 계산에서 요구되는 무결점 수준의 신뢰성을 멀티 에이전트 시스템으로 달성한 것은 주목할 만하다. Temperature=0 설정, 케이스별 독립 프롬프트, 자기 수정 루프 등 신중한 설계가 이를 가능하게 했다.
  3. Prompt Pool 기반 파라미터 스터디: 파일 수정 방식이 아닌 독립 프롬프트 생성 방식으로 다중 파라미터 변화를 처리한 접근은, OpenFOAM의 엄격한 구문 요구사항에 대한 효과적인 해결책이다.
  4. 자발적 해석해 비교: Post-processing Agent가 명시적 지시 없이도 Poiseuille 유동의 해석해를 자체적으로 구현하여 수치해와 비교한 점은 LLM의 물리 지식 활용 가능성을 보여준다.

Limitation & Further Study (한계 및 향후 연구)

한계

  1. 제한된 물리 복잡도: 테스트한 사례들이 비교적 표준적인 CFD 문제(층류, 단순 난류)에 한정되어 있으며, 산업 수준의 복잡한 문제(연소, 화학 반응, 유체-구조 상호작용 등)에 대한 검증이 없다.
  2. "잘 정의된 질의" 전제조건: 100% 성공률은 "well-formulated user queries"라는 전제 하에 달성되었으며, 모호하거나 불완전한 질의에 대한 로버스트니스는 검증되지 않았다.
  3. Claude-3.7-Sonnet 단일 모델 의존: 다른 LLM(GPT-4, DeepSeek 등)에서의 성능 비교가 본 논문에는 포함되지 않았다(선행 연구에서 일부 비교 존재).
  4. 3D 복잡 형상의 한계: snappyHexMesh를 통한 복잡 형상 처리가 시연되었으나, 산업 수준의 CAD 형상 처리에 대한 검증은 제한적이다.
  5. 계산 비용 미보고: LLM API 호출 비용, 시뮬레이션 실행 시간 등에 대한 분석이 없다.

향후 연구 방향

  • 산업 수준의 복잡한 CFD 문제로의 확장
  • 다중 물리(열전달, 연소, FSI 등) 시뮬레이션 지원
  • 시뮬레이션 결과에 기반한 자동 설계 최적화 루프 구현
  • 다른 계산 도메인(구조역학, 전자기학 등)으로의 아키텍처 확장

Evaluation

Evaluation

총평: 자연어-CFD 워크플로우 자동화를 end-to-end로 구현하고 100% 성공률을 실증한 실용적 연구로, 멀티에이전트 아키텍처와 OpenFOAM 통합이 유사 도구 대비 차별화된다.

# Agent Laboratory: Using LLM Agents as Research Assistants

- **저자**: Samuel Schmidgall, Yusheng Su, Ze Wang, Ximeng Sun, Jialian Wu, Xiaodong Yu, Jiang Liu, Michael Moor, Zicheng Liu, Emad Barsoum
- **소속**: AMD, Johns Hopkins University, ETH Zurich
- **발표**: arXiv preprint (2025)
- **DOI**: [10.48550/arXiv.2501.04227](https://doi.org/10.48550/arXiv.2501.04227)

---

## Essence (본질)

LLM 에이전트를 활용하여 연구의 전 과정(문헌 조사 -> 실험 -> 보고서 작성)을 자동화하는 프레임워크 **Agent Laboratory**를 제안한다. 인간 연구자가 연구 아이디어를 입력하면, PhD/Postdoc/ML Engineer/SW Engineer/Professor 등의 역할을 가진 전문화된 LLM 에이전트들이 파이프라인을 통해 코드 저장소와 연구 보고서를 생성한다. 완전 자율 모드와 인간 피드백을 반영하는 co-pilot 모드를 모두 지원하며, 기존 자율 연구 방법 대비 84% 비용 절감(gpt-4o 기준 논문당 $2.33)을 달성한다.

## Motivation (동기)

과학자들은 시간과 자원의 제약으로 탐구할 수 있는 연구 아이디어의 수가 제한되어, 많은 양질의 아이디어가 미탐구 상태로 남는다. 기존의 자율 연구 시스템(The AI Scientist 등)은 LLM이 독자적으로 아이디어를 생성하고 논문을 작성하는 방식으로, 인간 연구자의 의도와 괴리가 발생하고 비용이 높다. Agent Laboratory는 인간의 창의적 아이디어를 출발점으로 삼아 LLM이 저수준 코딩과 작성을 보조하는 "보완적(complementary)" 역할에 초점을 맞춘다.

## Achievement (성과)

1. **3단계 연구 파이프라인 구현**: 문헌 조사(arXiv API 기반), 실험(mle-solver를 통한 자율적 ML 코드 생성/최적화), 보고서 작성(paper-solver를 통한 LaTeX 보고서 생성)의 완전한 워크플로우.
2. **인간 평가 결과**: o1-preview 백엔드가 가장 높은 유용성(4.4/5)과 보고서 품질(3.4/5)을 달성. 인간 피드백을 포함한 co-pilot 모드에서 전체 점수가 자율 모드 대비 향상(NeurIPS 스타일 전체 점수 3.8 -> 4.38/10).
3. **비용 효율성**: gpt-4o 백엔드 기준 전체 워크플로우 비용 $2.33으로, The AI Scientist($15)의 1/6 수준.
4. **mle-solver 성능**: MLE-Bench 10개 Kaggle 챌린지에서 MLAB, OpenHands, AIDE 대비 가장 많은 메달(금 2, 은 1, 동 1) 획득. 10개 중 6개에서 인간 중앙값 이상 성능.
5. **높은 사용성**: co-pilot 모드의 사용성 4.0/5, 유틸리티 3.5/5, 지속 사용 의향 3.75/5.

## How (방법론)

- **멀티 에이전트 아키텍처**: PhD, Postdoc, ML Engineer, SW Engineer, Professor 역할의 에이전트가 단계별로 협업. 각 에이전트는 특화된 도구(arXiv API, Python 실행, HuggingFace 데이터셋 검색 등)를 사용.
- **mle-solver**: ML 코드를 자율적으로 생성/테스트/개선하는 모듈. EDIT(부분 수정)과 REPLACE(전체 재생성) 연산을 반복하며, LLM 기반 자기 평가(scoring)와 자기 반성(self-reflection)을 통해 성능을 향상. 상위 프로그램 풀(top program pool)을 유지하여 탐색-활용 균형을 관리.
- **paper-solver**: LaTeX 기반 학술 보고서를 scaffold -> arXiv 조사 -> 반복 편집 -> LLM 리뷰 순으로 생성. NeurIPS 가이드라인 기반의 자동 리뷰 시스템으로 품질을 평가.
- **co-pilot 모드**: 각 하위 과제 완료 시 인간 연구자가 피드백을 제공하거나 재실행을 요청할 수 있는 체크포인트 제공.

## Originality (독창성)

- **인간-AI 협업 패러다임**: 기존의 완전 자율 연구 시스템과 달리, 인간의 연구 아이디어를 입력으로 받아 인간의 창의성과 AI의 실행 능력을 결합하는 접근이 차별적.
- **역할 기반 멀티 에이전트 설계**: 연구 과정의 각 단계를 학술 조직의 역할(PhD, Postdoc, Professor 등)에 매핑한 설계가 직관적이고 독창적.
- **mle-solver의 트리 탐색 방식**: 프로그램 공간을 트리 탐색으로 탐색하며 상위 프로그램 풀을 유지하는 방식은 AIDE의 Solution Space Search를 일반화하여 연구 코드 전반에 적용.
- **자동 리뷰어 vs 인간 리뷰어 비교**: LLM 자동 리뷰어가 인간 리뷰어보다 평균 2.3점 높은 점수를 부여한다는 발견은 자동 평가의 한계를 실증적으로 보여줌.

## Limitation & Further Study (한계 및 향후 연구)

1. **자기 평가의 한계**: LLM 기반 자동 리뷰어가 인간 리뷰어보다 일관되게 높은 점수를 부여하여(평균 +2.3점), 자동 평가만으로는 실제 연구 품질을 정확히 반영하지 못함. NeurIPS 2024 평균 수락 점수(5.85/10)에 비해 인간 평가 점수가 평균 2.05점 낮음.
2. **할루시네이션 문제**: gpt-4o 등 일부 모델에서 실제 수행하지 않은 실험 결과를 보고서에 기술하는 할루시네이션 발생. 자율 연구 도구의 신뢰성에 심각한 위험.
3. **ML 분야 한정**: 현재 프레임워크가 머신러닝 연구에 특화되어 있어, 실험 과학(화학, 생물학, 물리학 등)이나 이론적 연구로의 확장이 필요.
4. **도구 사용의 제한**: 이미지/그래프 생성 기능이 제한적이며, 리포지토리 수준의 코드 관리 불가. 문헌 조사 단계에서 instruction-following 실패율이 높음(20-40%).
5. **co-pilot 모드의 역설**: 인간 피드백이 전체 점수를 향상시키지만, 에이전트의 의도를 인간의 비전에 맞추는 데 어려움이 있어 사용자 만족도가 자율 모드보다 오히려 낮은 항목 존재.
6. **재현성**: 연구 주제에 따른 성능 변동이 크며(예: image noise 주제에서 gpt-4o vs o1-mini 실험 품질 1.5 vs 4.0), 결과의 일반화 가능성 검증이 필요.

## Evaluation (총평)

Agent Laboratory는 LLM 에이전트를 연구 보조 도구로 활용하는 **실용적이고 체계적인 프레임워크**이다. 인간의 연구 아이디어를 출발점으로 삼아 저수준 구현을 자동화한다는 설계 철학이 현실적이며, gpt-4o 기준 논문당 $2.33의 비용 효율성은 주목할 만하다. mle-solver의 MLE-Bench 성능은 자동 ML 코드 생성의 가능성을 보여준다. 그러나 NeurIPS 스타일 인간 평가 점수가 3.5-4.4/10 수준에 머무르고, 할루시네이션 문제가 해결되지 않은 점은 실질적 연구 도구로서의 성숙도가 아직 부족함을 시사한다. 자동 리뷰어와 인간 리뷰어 간의 괴리를 실증적으로 밝힌 것은 커뮤니티에 유용한 기여이다. ML 이외의 과학 분야로의 확장과 할루시네이션 감지/방지 메커니즘 강화가 향후 핵심 과제이다.

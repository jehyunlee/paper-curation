Galactica: A Large Language Model for Science

Figure 1

Essence

과학 문헌과 데이터의 폭발적 증가로 인한 정보 과부하 문제를 해결하기 위해, 과학 지식을 저장·결합·추론할 수 있는 대규모 언어 모델 Galactica를 제안한다. 4,800만 편의 논문, 교과서, 강의 노트, 수백만 개의 화합물 및 단백질 서열, 과학 웹사이트, 백과사전 등 총 1,060억 토큰의 고품질 큐레이션 과학 코퍼스로 학습하여, 기존 범용 LLM과 차별화된 과학 특화 언어 모델을 구현하였다. 검색 엔진 중심의 기존 과학 지식 접근 방식을 넘어, 언어 모델이 과학의 새로운 인터페이스가 될 수 있음을 보여주는 선구적 연구이다.

Motivation (동기)

  • 과학 문헌은 기하급수적으로 증가하고 있으며(2022년 5월 기준 arXiv에 하루 평균 516편 제출), 단일 연구자가 모든 관련 논문을 읽는 것은 불가능하다.
  • 현재 과학 지식 접근 방식인 검색 엔진은 지식을 직접 조직하지 못하고, Wikipedia·UniProt·PubChem 등 2차 계층에 의존하며, 이는 비용이 많이 드는 인간 기여를 필요로 한다.
  • 기존 범용 LLM들은 비큐레이션된 크롤 기반 코퍼스(예: PaLM의 50%가 소셜 미디어 대화)를 사용하여 과학 작업에는 비효율적이다.
  • 과학 텍스트의 고유한 특성(수학 표현, 화학식, 단백질 서열 등)을 효과적으로 처리하는 전문 모델이 필요하다.

Achievement (성과)

  • 기술 지식 프로브: LaTeX 방정식에서 GPT-3 대비 월등한 성능 (68.2% vs 49.0%)
  • 수학적 추론: Mathematical MMLU에서 Chinchilla 대비 우수 (41.3% vs 35.7%), MATH 벤치마크에서 PaLM 540B 대비 월등 (20.4% vs 8.8%, 파라미터는 18배 적음)
  • 다운스트림 과학 QA: PubMedQA (77.6%, SOTA), MedMCQA dev (52.9%, SOTA) 달성
  • 범용 벤치마크: BIG-bench 57개 과제에서 OPT-175B, BLOOM-176B보다 더 작은 모델(120B)로 우수한 성능
  • 인용 예측: 튜닝된 sparse/dense retrieval 방법보다 뛰어난 인용 예측 정확도
  • 화학 이해: IUPAC 명명법 자기 지도 학습 (120B에서 39.2% 정확도)
  • 단백질 기능 예측: 서열로부터 기능 키워드 및 설명 예측 능력 (스케일에 따라 성능 향상)
  • 독성/편향: 기존 LLM 대비 현저히 낮은 독성 및 편향 (CrowS-Pairs 60.5%, StereoSet ICAT 65.6%)

How (방법론)

1. 큐레이션된 과학 코퍼스

  • 1,060억 토큰: 논문 83%, 코드 6.9%, 참고자료 6.5%, 지식 베이스 2%, 필터링된 CommonCrawl 1%
  • 모든 데이터를 공통 마크다운 형식으로 처리하여 소스 간 지식 융합

2. 특화 토큰화 (Specialized Tokenization)

  • 인용 토큰: [START_REF]/[END_REF]로 인용을 감싸 암묵적 인용 그래프 표현
  • 작업 메모리 토큰 : 단계별 추론을 내부 작업 메모리처럼 감싸는 토큰 — chain-of-thought의 한계를 보완
  • SMILES 토큰: [START_SMILES]/[END_SMILES]로 화학식 문자 단위 토큰화
  • 아미노산/DNA 서열 토큰: 문자 단위 토큰화로 생물학적 서열 처리
  • 수학/숫자: 개별 문자/숫자를 독립 토큰으로 분리

3. 프롬프트 사전학습 (Prompt Pre-Training)

  • 다양한 ML 학습 데이터셋을 텍스트 프롬프트로 변환하여 일반 코퍼스와 함께 사전학습에 포함
  • fine-tuning 없이도 QA·요약 등 다운스트림 과제에서 즉시 활용 가능

4. 모델 아키텍처

  • Transformer 기반, 125M ~ 120B까지 5개 스케일
  • GeLU 활성화, ALiBi 위치 인코딩(2048 컨텍스트 윈도우), BPE 토크나이저(50,000 어휘)
  • 반복 에폭 학습 가능 (큐레이션 코퍼스의 높은 품질 덕분에 과적합 없음)

Originality (독창성)

  1. 큐레이션 패러다임의 복권: 비큐레이션 크롤 기반 학습이 주류인 상황에서, 고품질 과학 코퍼스만으로 범용 LLM을 능가할 수 있음을 증명 — "데이터 품질 > 데이터 양" 관점 제시
  2. 작업 메모리 토큰: chain-of-thought prompting의 한계(프롬프트 발견 의존성, 중간 단계 누락)를 보완하는 새로운 추론 메커니즘. Python 스크립트 오프로딩을 통한 모델-기계 공생 가능성 제시
  3. 멀티모달 과학 언어 모델: 자연어, LaTeX, SMILES, 아미노산 서열, DNA 서열, 코드를 단일 모델에서 통합 처리하는 "Tokenizing Nature" 접근법
  4. 인용 예측의 새로운 패러다임: 검색 기반 접근을 넘어 언어 모델의 문맥 연상 능력으로 인용을 예측하고, 스케일에 따라 인용 분포가 실제 분포에 수렴함을 보임

Limitation & Further Study (한계 및 향후 연구)

한계

  • 코퍼스 제약: 오픈 액세스 자료만 사용하여 폐쇄형 논문·교과서의 지식이 누락됨. 화합물(~200만/1.1억)과 단백질도 일부만 포함
  • 코퍼스 효과 vs 프롬프트 효과: 사전학습 프롬프트와 과학 코퍼스의 개별 기여도를 분리하지 않음
  • 인용 편향: 대규모 모델에서도 인기 논문에 대한 편향이 잔존
  • 일반 지식: 범용 사회적 지식에는 부적합 (의도적 트레이드오프)
  • 텍스트 한계: 3D 좌표 등 기하학적 정보를 표현하지 못함 (화학 구조의 근본적 제약)
  • Hallucination: 여전히 존재하며, 생성 결과의 신뢰성 보장 메커니즘 부재
  • 컨텍스트 윈도우: 2,048 토큰으로 긴 과학 문서 처리에 제한적

향후 연구

  • 혼합 디노이징 학습(mixture-of-denoising) 목적함수 도입
  • 컨텍스트 윈도우 확장 (긴 문서·긴 단백질 서열 처리)
  • 이미지 모달리티 통합
  • 토큰의 프롬프트 다양성 확대
  • 검증(verification) 계층 개발
  • 지속 학습(continual learning) 전략 수립
  • 검색 증강(retrieval augmentation)과의 결합

Evaluation

Evaluation

총평: 과학 지식에 특화된 106B 토큰 대규모 언어 모델로 수식·코드·인용을 통합 처리하는 Meta AI의 야심찬 연구로, 조기 철회 논란에도 불구하고 과학 전용 LLM 연구의 이정표가 되었다.
